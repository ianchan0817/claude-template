# Explain

When explaining code:

1. Start with the **purpose** — what problem does this solve?
2. Describe the **inputs and outputs** concisely.
3. Walk through the **key logic steps** in plain language.
4. Call out any **non-obvious decisions** or gotchas.
5. Tailor the depth to what the user seems to already know.

Do not just restate the code. Explain the *why*, not just the *what*.
